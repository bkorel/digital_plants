# Модель мира

## Сетка

- Размер: `W × H` (по умолчанию **300×200**)
- `SOIL_Y` — граница почвы (по умолчанию **124**)
- Выше `SOIL_Y` — воздух, ниже — почва

## Свет и затенение

Базовая освещённость растёт с высотой (`altitudeLight`): у земли света меньше,
у вершины мира — максимум. Для колонки `x` на высоте `y`:

```
light = altitudeLight(y) * (1 - SHADE_PER_LAYER)^effectiveLayers
если effectiveLayers > MAX_SHADE_LAYERS → light = 0
```

`effectiveLayers` — число слоёв растений выше в той же колонке с учётом
**рассеивания**: ниже последней клетки-затенителя каждые `SHADE_SCATTER_INTERVAL`
(20) клеток эффективная тень делится пополам.

Под толстым пологом (более `MAX_SHADE_LAYERS` эффективных слоёв) света нет —
фотосинтез под кроной невозможен.

**STEM** (ствол) даёт `STEM_PHOTO_GAIN_FACTOR` (20%) фотосинтеза от **SPROUT**
(мерistemы/лист).

### Затенённая мерistema

Если над мерistemой (`SPROUT` в воздухе) больше `SHADED_SPROUT_LAYERS` (5) слоёв
тени, по геному (`genomeShadeSenescence`):

- **лигнификация** — клетка становится `STEM`;
- **минерализация** — клетка снимается, энергия уходит в почву.

После потери листьев проверяется связность: ветки без пути к почве отваливаются.

## Вода

```
water(depth) = min(1, (depth+1) * WATER_PER_DEPTH)
```

В воздухе влаги мало (`AIR_WATER = 0.05`) — фотосинтез невозможен без корневого
водоснабжения; в почве вода растёт с глубиной. Снабжение растения водой
(`plantWaterSupply`) берётся от его корней (или ростка прямо над почвой).

## Минеральный цикл

Поле `M[x][y]` в почве.

### Старт

```
M0(d) = MINERAL_BASE + MINERAL_DEPTH_GAIN * d
```

### Порядок в tick()

1. **Добыча корнями** — `uptake = min(M, UPTAKE_MAX)`, списание из `M`, зачисление в `cellEnergy` корня
2. **Доход/расход/транспорт** растений
3. **Рост VM**
4. **Семена** — падение, прорастание
5. **Смерть** — осаждение `DEATH_ENERGY_RETURN * cellEnergy` в почву
6. **Диффузия** — просачивание вниз + латеральное выравнивание

### Просачивание, диффузия, регенерация

```
down = M[x][y] * MINERAL_SINK_RATE
M[x][y] -= down; M[x][y+1] += down
flowLR = MINERAL_DIFFUSE_RATE * ((M[x-1] + M[x+1])/2 - M[x][y])
# восстановление к базовому уровню (если ниже него):
M += (baseline - M) * MINERAL_REGEN_RATE
```

На нижней границе `down` уходит из мира (сток). Регенерация медленно
возвращает выеденный приповерхностный слой к естественному уровню — без неё
популяция вымирает после первого бума.

## Энергия растений

- У каждой клетки: `cellEnergy`, ёмкость `CAP[type]`
- Фотосинтез только в надземных «листьях» (`SPROUT` и `STEM` выше почвы; у
  `STEM` — `STEM_PHOTO_GAIN_FACTOR`). По закону Либиха ограничен минимумом из
  света и минерального снабжения корней:
  `min(light, mineralSupply/MINERAL_SUPPLY_NORM) * PHOTO_GAIN_FACTOR * (1 - ratio)`
- Корни: добыча минералов идёт на минеральное снабжение (множитель фотосинтеза),
  не напрямую в энергию.
- Поддержание: `MAINTAIN[type]` за тик; после `SENESCENCE_START` × maxAge
  расход растёт (долгожители «стареют» дороже)
- Транспорт по соседним клеткам: `TRANSPORT_RATE * (pot_a - pot_b)`

### Ограничение размера

Жёсткого лимита клеток нет. Размер ограничивается балансом:

```
доход ≈ фотосинтез верхушки (закон Либиха)
расход ≈ Σ MAINTAIN[type] × senescence(age)
```

Ростовой бюджет за тик:

```
surplus = totalEnergy - upkeep × GROW_ENERGY_RESERVE_TICKS - GROW_MIN_ENERGY_RESERVE
budget = min(MAX_GROW_ACTIONS_CAP,
             MAX_GROW_ACTIONS_PER_PLANT_PER_TICK
               + floor(surplus / ENERGY_PER_EXTRA_GROW_ACTION))
```

Крупное растение без достаточной кроны уходит в минус и гибнет (`energy < 0.5`).

### Короткоживущие vs долгожители

| Стратегия | maxAge (геном) | Семена | Рост |
|-----------|----------------|--------|------|
| Быстрая | 30–80 | рано (`HEIGHT`, низкий порог) | компактная крона, мало upkeep |
| Долгая | 120–220 | поздно (`AGE` > ~0.5 жизни) | глубокие корни, много `STEM`, накопление энергии |

Обе стратегии зашиты в шаблон генома; отбор подстраивает пороги мутациями.

## Смерть

- `energy < 0.5` или `age >= genomeMaxAge(genome)`
- Только `SEED`-клетки → семена в почве (с мутацией)
- Остальные (`ROOT`/`STEM`/`SPROUT`) → перегной (минералы в почву)

Размножение возможно лишь через `SEED`-операцию с надземного побега
(не ниже `MIN_SEED_HEIGHT`) — «посев на смерти» закрыт.

## Семена и прорастание

При сбросе семя ищет свободную ячейку на **поверхности** почвы (`SOIL_Y`) не
дальше `SEED_SCATTER` клеток по X от колонки создания; если рядом места нет —
погибает (не улетает далеко и не закапывается глубоко). Семя ждёт `SEED_GERMINATION_TICKS`, затем при свободной ячейке и
`energy >= MIN_SEED_ENERGY` может прорасти (с вероятностью, лимит
`MAX_GERMINATIONS_PER_TICK`). Пока `ticksInSoil <= SEED_GERMINATION_TICKS`, семя
**не тратит** энергию; после окна, если проростка нет — `SEED_SOIL_UPKEEP` за тик.
При `energy < MIN_SEED_ENERGY` семя **сразу** исчезает (не ждёт обнуления). Геном задаёт долю энергии при сбросе (`SEED` + аргумент 0..1).
Прорастание принудительно происходит на уровне почвы (`SOIL_Y`), новый проросток получает небольшой разброс начального
возраста (рассинхронизация гибели когорты).

Проросшее растение занимает «место» у соседей: семена в колонках
±`GERMINATION_NEIGHBOR_BLOCK` по X не могут прорасти, пока рядом есть живое
растение с клеткой на поверхности или в почве у поверхности.

## Занятость клеток

- Одна ячейка — один объект: растение **или** семя в почве, никогда оба.
- Семена в почве резервируют ячейку в `occupancy` (метка `SEED_OCC`).
- В воздухе клетки **разных** растений могут соприкасаться. Конкуренция кроны
  разрешается на следующем тике (см. ниже). Стартовые посадки разнесены на
  `MIN_PLANT_SPACING` клеток по X.

## Конкуренция листьев

После дохода/расхода за тик, **до роста VM**:

| Контакт (разные растения, воздух) | Исход |
|-----------------------------------|--------|
| лист (`SPROUT`) — лист | погибает лист с **меньшей** энергией (при равенстве — с меньшим id) |
| лист — ствол (`STEM`) | погибает **лист** |
| ствол — ствол | без эффекта |

После гибели листьев проверяется связность: надземные клетки без пути к почве
(`y >= soilY`) отваливаются и превращаются в минералы (`DEATH_ENERGY_RETURN`).
Оставшаяся часть растения продолжает жить и расти.

## Шипы

Клетки `SPIKE` не растут (не исполняют VM). Геном выставляет их через:

- **`SPIKE`** — шип в соседней клетке `dir`; родитель остаётся меристемой.
- **`SHOOT`** — шип на **2 клетки** в `dir` (промежуточная клетка должна быть
  свободным воздухом); завершает прогон меристемы.

Чужие листья (`SPROUT` в воздухе) в радиусе `SPIKE_LEAF_KILL_RADIUS` (манхэттен)
от шипа гибнут на тике конкуренции кроны — без прямого соприкосновения.

## Корни и побеги

- Тип зрелой ткани задаётся **положением в мире**, не геномом: в почве (`y >= soilY`)
  — `ROOT`, в воздухе — `STEM`.
- Когда меристема (`SPROUT`) даёт потомка (`GROW` или `BRANCH`), она созревает
  в ткань по этому правилу. **Ниже поверхности** (`y > soilY`) после ответвления
  родительская клетка становится корнем; на линии почвы (`y === soilY`) при
  `BRANCH` остаётся меристемой, чтобы можно было пустить побег вверх. В воздухе
  при `BRANCH` меристема тоже сохраняется (несколько боковых побегов за тик).
- Активный кончик роста в почве остаётся `SPROUT` до следующего действия или
  созревания «в ткань», если программа больше не срабатывает.

## Цикл tick() (сводка)

```
light → uptake → income/upkeep → age → foliage → growth → transport → seeds → death → diffuse → light
```
