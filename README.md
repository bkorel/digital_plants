# Digital Plants Evolution

Веб-симуляция эволюции **цифровых растений**: 2D-мир с ресурсами (свет, вода, минералы), геномом-«ДНК» и виртуальной машиной роста. Растения конкурируют за энергию, отбрасывают семена, мутируют и умирают, возвращая часть энергии в почву.

## Стек

- **TypeScript** — ядро симуляции (`src/sim/`)
- **React 19 + Vite** — интерфейс (`src/ui/`)
- **Canvas 2D** — отрисовка мира

## Быстрый старт

```bash
npm install
npm run dev
```

Откройте адрес из терминала (обычно `http://localhost:5173`).

### Сборка и предпросмотр

```bash
npm run build
npm run preview
```

Артефакты попадают в `dist/`.

### Публикация на GitHub Pages

1. В [vite.config.ts](vite.config.ts) задан `base: './'` — относительные пути для статического хостинга.
2. В репозитории: **Settings → Pages → Build and deployment → GitHub Actions**.
3. После **merge PR в `main`** workflow [.github/workflows/deploy-pages.yml](.github/workflows/deploy-pages.yml) собирает `dist/` и публикует сайт.

Прямой push в `main` не используется — только через Pull Request и code review.

#### Защита ветки `main` (один раз в GitHub)

**Settings → Branches → Add branch protection rule** для `main`:

- **Require a pull request before merging** — включить
- **Require approvals** — минимум 1
- **Require status checks to pass** — включить, выбрать job **build** из workflow **CI**
- **Do not allow bypassing the above settings** — включить (если доступно)
- **Restrict who can push to matching branches** — опционально, только maintainers

Перед merge PR должен пройти [.github/workflows/ci.yml](.github/workflows/ci.yml) (`npm ci` + `npm run build`).

Локально: `npm run build`, затем содержимое `dist/` можно залить вручную (ветка `gh-pages` и т.п.).

## Режимы приложения

| Режим | Описание |
|-------|----------|
| **Эволюция** | Полный мир: 30 стартовых растений, семена, конкуренция, мутации |
| **Лаборатория** | Изолированный образец: один геном, без посторонней конкуренции; снимок эволюции сохраняется при входе |

## Управление

| Действие | Описание |
|----------|----------|
| **Пауза / Продолжить** | Остановить или возобновить симуляцию |
| **Шаг** | Один тик (также **Пробел**, если фокус не в поле ввода) |
| **Рестарт** | Новый мир с текущим сидом (в лаборатории — перезапуск образца) |
| **Скорость** | Сколько тиков выполняется за кадр (~14 ms бюджет на кадр) |
| **Сид** | Зерно ГПСЧ для воспроизводимости (только в эволюции) |
| **Клик по растению** | Выбор, пауза, режим трассировки |
| **Коллекция** | Сохранённые геномы для подсева; пресеты «Стрелок» и «Стеснитель» |

### Режимы отображения

| Режим | Что показывает |
|-------|----------------|
| **Растения** | Цвет по геному |
| **Энергия** | Заполнение клеток; минералы в почве |
| **Анатомия** | Типы тканей (корень / ствол / мерistema / семя / шип) |
| **Потоки** | Направление и сила перетока энергии между клетками |
| **Трассировка** | События последнего тика выбранного растения |

## Мир (текущие параметры)

- Сетка **300×200** клеток, поверхность почвы **`SOIL_Y = 124`**
- Небо: градиент освещённости + затенение кроной (`MAX_SHADE_LAYERS = 4`)
- **Рассеивание тени**: ниже последнего затенителя в колонке каждые **20** клеток эффективная тень делится пополам
- **Фотосинтез ствола (STEM)**: **20%** от мерistemы (SPROUT)
- **Затенённая мерistema**: при **>5** слоях тени — лигнификация или минерализация (зависит от генома)
- **Двойной рост**: одно `GROW` может вытянуть **2** клетки за **3×** стоимости
- Семена падают на поверхность почвы с разбросом до **3** клеток по X
- Стартовые и проросшие растения разнесены минимум на **5** клеток

Подробнее — в [docs/world.md](docs/world.md) и панели «Правила мира» в UI.

## Основные механики

- **Геном** — массив байт; каждый байт декодируется в инструкцию VM (`GROW`, `BRANCH`, `SEED`, `SPIKE`, `SHOOT`, сенсоры, условия). См. [docs/genome.md](docs/genome.md).
- **Энергия** — в клетках; течёт от листьев и корней к мерistemам; фотосинтез по закону Либиха (свет × минералы).
- **Минералы** — динамическое поле в почве: добыча корнями, перегной при смерти, просачивание, диффузия, медленная регенерация.
- **Конкуренция кроны** — соседние листья разных растений; аура шипов гибнет чужие побеги в радиусе; выстрелы SHOOT; отвал незакреплённых веток.
- **Эволюция** — мутации при прорастании семян, отбор по свету и ресурсам.

## Архитектура

```
src/
  sim/              # Чистая логика (без React)
    config.ts       # Константы баланса
    types.ts        # Типы
    rng.ts          # Сидируемый ГПСЧ
    genome.ts       # Геном, мутации, отпечаток цвета
    environment.ts  # Свет, вода, минералы
    plant.ts        # Клетки, энергия, VM роста
    foliage.ts      # Конкуренция кроны, затенённая мерistema
    world.ts        # Цикл tick(), мир
    plantEvents.ts  # События тика для трассировки
  ui/               # React-компоненты
    App.tsx         # Цикл симуляции (rAF, бюджет ~14 ms)
    WorldCanvas.tsx # Canvas: кэш слоёв фона + отрисовка растений
scripts/            # Утилиты проверки баланса (simcheck, fecundity, …)
docs/               # Подробная документация
```

### Цикл симуляции

```
uptake → income/upkeep → age → foliage → light → growth VM → transport → seeds → death → diffuse → light
```

### Производительность

- Тики: батч в `requestAnimationFrame` с бюджетом ~14 ms; перерисовка UI только при изменении `tickCount`.
- Симуляция: один `rebuildLight()` после foliage и после diffuse; `Map` для uptake; мёртвые растения удаляются из массива.
- Canvas: **кэш слоя почвы** (статичный градиент) и **кэш неба** (пересчёт при смене тика); отдельный rAF только пока есть падающие семена.
- События трассировки (`tickEvents`) собираются **только** в режиме TRACE для выбранного растения.
